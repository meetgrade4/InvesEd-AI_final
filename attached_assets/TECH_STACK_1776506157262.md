# InvesEd AI вЂ” Tech Stack Specification

## App Identity
- **Name:** InvesEd AI
- **Tagline:** InvestSim for Teens
- **Type:** Web app (React SPA) вЂ” mobile-responsive
- **Colour Palette:** Primary `#1E3A5F`, Accent `#2E86AB`, Light `#E8F4F8`, Success `#1B6B3A`

---

## Frontend

| Layer | Technology | Notes |
|---|---|---|
| Framework | React 18 + TypeScript | Functional components only, no class components |
| Styling | TailwindCSS | Use JIT mode, no custom CSS files unless unavoidable |
| Routing | React Router v6 | `createBrowserRouter`, nested routes |
| Animations | Framer Motion | Page transitions, badge reveals, XP bursts, chart draw-ins |
| Lottie | `lottie-react` | For JSON animations: level-up, certificate reveal, coin SIP |
| Charts | Recharts | Portfolio pie, line charts, bar charts вЂ” all Indian data |
| State | React Context + useState | No Redux вЂ” Context for auth/user, local state for UI |
| Forms | React Hook Form | Onboarding quiz, sign-up, trade entry |
| Icons | Lucide React | Consistent icon set throughout |
| Toasts | react-hot-toast | XP gain, badge earned, AI alerts |
| Date | date-fns | Format Indian date conventions (DD/MM/YYYY) |

---

## Backend / Services

| Service | Technology | Notes |
|---|---|---|
| Authentication | Firebase Auth | Google OAuth + Email/Password + Phone OTP |
| Database | Firebase Firestore | NoSQL вЂ” users, portfolios, trades, progress |
| File Storage | Firebase Storage | Certificates, avatar uploads |
| AI Chatbot | Anthropic Claude API (`claude-sonnet-4-6`) | Context-aware, portfolio-injected system prompt |
| Market Data | Mock JSON (demo) | `/src/data/marketData.ts` вЂ” 50 stocks + 20 MFs with realistic prices |
| Hosting | Replit (dev) в†’ Firebase Hosting (prod) | |

## Key Conventions

- **Currency:** Always display as `в‚№X,XX,XXX` (Indian numbering). Use `formatINR()` from `utils/formatters.ts`
- **Dates:** DD/MM/YYYY throughout. Never MM/DD/YYYY.
- **Numbers:** Lakhs and crores, not millions and billions. `formatIndianNumber()` utility.
- **Colours:** Always use Tailwind custom colours mapped to the brand palette вЂ” never hardcode hex in JSX.
- **Animations:** Every XP gain в†’ floating `+XP` toast. Every badge в†’ full-screen reveal. Every level-up в†’ Lottie burst.
- **Mobile-first:** All layouts start at 375px (mobile) and scale up. Use `sm:`, `md:`, `lg:` breakpoints.
- **Accessibility:** All interactive elements must have `aria-label`. Quiz timers must use `aria-live="polite"`.
